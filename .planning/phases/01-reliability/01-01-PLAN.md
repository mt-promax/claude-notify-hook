  <action>
Replace the existing spawn block (the try/catch at lines ~116-128) with a version that captures
spawn failures and PowerShell stderr to the error log. Addresses RELY-02.

Step 1 - Add imports alongside the existing child_process import at the top of the file:

  const path = require("path");

Step 2 - Add logPath constant immediately after the process.on("uncaughtException") line:

  const logPath = path.join(process.env.TEMP || process.env.USERPROFILE, "claude-notify-error.log");

Use process.env.TEMP first (standard Windows %TEMP% dir, produces %TEMP%\claude-notify-error.log as
required by RELY-02 success criteria). Fall back to USERPROFILE only if TEMP is unset.

Step 3 - Replace the existing spawn try/catch block entirely:

  const encoded = Buffer.from(balloon, "utf16le").toString("base64");
  try {
    const ps = spawn("powershell.exe", [
      "-WindowStyle", "Hidden",
      "-NonInteractive",
      "-EncodedCommand", encoded
    ], {
      detached: true,
      windowsHide: true,
      stdio: ["ignore", "ignore", "pipe"]
    });

    ps.on("error", (err) => {
      try {
        const ts = new Date().toISOString();
        fs.appendFileSync(logPath, "[" + ts + "] SPAWN_ERROR: " + err.message + "\n");
      } catch (_) {}
    });

    let errBuf = "";
    ps.stderr.on("data", (chunk) => { errBuf += chunk.toString("utf8"); });
    ps.stderr.on("close", () => {
      if (errBuf.trim()) {
        try {
          const ts = new Date().toISOString();
          fs.appendFileSync(logPath, "[" + ts + "] POWERSHELL_ERROR:\n" + errBuf + "\n");
        } catch (_) {}
      }
    });

    ps.unref();
  } catch (_) {}

Key constraints:
- stdio changes from "ignore" to ["ignore", "ignore", "pipe"]. Stdout stays ignored, stderr is piped.
- All fs.appendFileSync calls are in inner try/catch blocks. Logging failure must never propagate.
- ps.unref() is still called. Hook exits immediately; stderr pipe stays open until PowerShell exits.
- The outer try/catch remains as a safety net for synchronous spawn failures.
- Keep "const encoded = ..." in its existing position (before the try block).
- Do NOT touch the execFile sound block at the top. Phase 3 scope.

<task type="auto">
  <name>Task 2: Add balloon stabilization delay and BalloonTipShown handler to PowerShell template</name>
  <files>hooks/notify-waiting.js</files>
  <action>
Modify the PowerShell balloon script template (the multi-line string assigned to const balloon) to add
two reliability improvements on the PowerShell side. Addresses RELY-03 and RELY-01.

Change 1 - Add 100ms stabilization delay (RELY-03).

Find the line: $n.Visible = $true
Find the line: $n.ShowBalloonTip(6000, ...
Insert between them, after $n.Visible = $true and before $n.ShowBalloonTip:

  # RELY-03: stabilization delay prevents silent balloon suppression on shell registration race
  Start-Sleep -Milliseconds 100

This prevents the race condition from PITFALLS.md Pitfall 3: ShowBalloonTip called before Windows
shell has registered the icon in the notification area is silently dropped. 100ms is the documented
safe value (50ms is sometimes insufficient on fast machines).

Change 2 - Add BalloonTipShown event handler (RELY-01, RELY-03).

Add this handler alongside the other event handlers, BEFORE $n.ShowBalloonTip is called:

  # RELY-03: confirm balloon appeared - writes to log for diagnostics
  $n.add_BalloonTipShown(({
      try {
          $ts = Get-Date -Format o
          Add-Content -Path "$env:TEMP\claude-notify-error.log" -Value "[$ts] BalloonTipShown: notification appeared"
      } catch {}
  }).GetNewClosure())

Required ordering inside the balloon template after changes:
  1. Add-Type blocks (unchanged)
  2. Get-ParentPid function (unchanged)
  3. PID walk loop + fallback name search (unchanged)
  4. NotifyIcon creation: $n.Visible = $true (unchanged)
  5. [NEW] Start-Sleep -Milliseconds 100
  6. BalloonTipClicked handler (unchanged)
  7. BalloonTipClosed handler (unchanged)
  8. [NEW] BalloonTipShown handler
  9. $n.ShowBalloonTip(...) call - now after stabilization delay and all handler registrations
  10. Timer + Application.Run() (unchanged)

CRITICAL: BalloonTipShown handler MUST be registered before ShowBalloonTip is called. If registered
after, the event may fire before the handler attaches and the log write is skipped.

Use $env:TEMP in the PowerShell handler (not a hardcoded path). This resolves to the same directory
as process.env.TEMP in Node.js, so both sides write to the same log file.

The try/catch inside the handler swallows all exceptions. Logging must never abort the balloon process.
